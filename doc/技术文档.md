# V-Calendar 智能语音日历系统技术文档

## 1. 项目概述

V-Calendar（Valendar）是一款基于 Electron + Vue 3 + TypeScript 构建的**智能语音日历闹钟系统**，专为 Windows 桌面平台设计。项目采用**纯客户端、无后端服务**的架构，所有核心逻辑、数据存储、AI 推理均在本地客户端完成。

### 1.1 核心特性

- **多视图日历展示**：支持月视图、周视图、日视图三种展示模式
- **农历与节假日集成**：使用 `chinese-days` 库提供农历日期、二十四节气、法定节假日查询
- **智能事件管理**：支持 CRUD 操作、重复事件、事件分类、闹钟提醒
- **AI 语音交互**：集成自然语言理解（NLU）和自动语音识别（ASR）技术
- **对话式日程解析**：通过语音或文字输入，AI 自动解析并执行日程操作
- **本地数据持久化**：使用 `electron-store` 实现可靠的数据存储和备份

### 1.2 技术栈

| 层级 | 技术选型 |
|------|---------|
| 桌面框架 | Electron 39.x |
| 前端框架 | Vue 3.5 + Composition API |
| 类型系统 | TypeScript 5.x |
| 构建工具 | electron-vite 5.x + Vite 7.x |
| 状态管理 | Pinia 3.x |
| AI 框架 | LangChain / LangGraph |
| 农历/节假日 | chinese-days |
| 日期处理 | dayjs |
| 本地存储 | electron-store |
| 代码规范 | ESLint + Prettier |

## 3. 核心模块详解

### 3.1 主进程模块 (Main Process)

主进程负责管理系统窗口、IPC 通信、闹钟定时器、数据存储和系统通知。

#### 3.1.1 入口文件 (src/main/index.ts)

**职责**：
- 创建 BrowserWindow 窗口实例
- 注册所有 IPC 事件处理器
- 初始化存储服务
- 启动所有闹钟

**核心代码逻辑**：

```typescript
// 窗口配置
const mainWindow = new BrowserWindow({
  width: 1200,
  height: 800,
  minWidth: 900,
  minHeight: 600,
  show: false,
  autoHideMenuBar: true,
  webPreferences: {
    preload: join(__dirname, '../preload/index.js'),
    sandbox: false
  }
})

// 窗口就绪后启动闹钟服务
mainWindow.on('ready-to-show', async () => {
  mainWindow.show()
  alarmService.setMainWindow(mainWindow)
  await alarmService.startAllAlarms()
})
```

#### 3.1.2 存储服务 (src/main/services/storage.service.ts)

**职责**：
- 使用 `electron-store` 管理数据持久化
- 提供事件、闹钟、设置的 CRUD 操作
- 自动备份管理（最多保留 5 个备份）
- 数据导入导出

**数据结构**：

```typescript
interface StoreSchema {
  events: CalendarEvent[]    // 事件列表
  settings: Settings         // 应用设置
  alarms: Alarm[]            // 闹钟列表
}
```

**备份机制**：
- 在创建、更新、删除事件前自动创建备份
- 备份文件命名：`backup-{ISO时间戳}.json`
- 存储位置：`userData/backups/`
- 最多保留 5 个备份，自动清理旧备份

**事件验证规则**：
- 标题不能为空，不超过 200 字符
- 日期范围有效（开始日期 ≤ 结束日期）
- 时间格式必须为 HH:mm
- 同一天内结束时间必须晚于开始时间
- 描述不超过 2000 字符

#### 3.1.3 闹钟服务 (src/main/services/alarm.service.ts)

**职责**：
- 管理所有闹钟定时器
- 触发闹钟时发送系统通知
- 支持贪睡和取消操作
- 与渲染进程通信

**核心功能**：

```typescript
class AlarmService {
  private alarms: Map<string, NodeJS.Timeout> = new Map()
  private mainWindow: BrowserWindow | null = null

  // 启动所有有效闹钟
  async startAllAlarms(): Promise<void>

  // 调度单个闹钟
  scheduleAlarm(alarm: Alarm): void

  // 触发闹钟
  private triggerAlarm(alarm: Alarm): void

  // 贪睡
  async snoozeAlarm(alarmId: string, minutes: number): Promise<void>

  // 取消闹钟
  async dismissAlarm(alarmId: string): Promise<void>
}
```

**触发流程**：
1. 计算触发延迟时间
2. 设置 setTimeout 定时器
3. 定时器到期后显示系统通知
4. 通过 IPC 通知渲染进程显示弹窗

#### 3.1.4 IPC 事件处理

| 模块 | 事件名 | 描述 |
|------|--------|------|
| 事件 | `event:getAll` | 获取所有事件（支持过滤） |
| 事件 | `event:get` | 获取单个事件 |
| 事件 | `event:create` | 创建事件（同时创建闹钟） |
| 事件 | `event:update` | 更新事件（重建闹钟） |
| 事件 | `event:delete` | 删除事件（同时删除闹钟） |
| 事件 | `event:deleteMany` | 批量删除 |
| 事件 | `event:search` | 搜索事件 |
| 事件 | `event:getByDateRange` | 按日期范围查询 |
| 事件 | `event:getStats` | 获取统计数据 |
| 闹钟 | `alarm:getAll` | 获取所有闹钟 |
| 闹钟 | `alarm:snooze` | 贪睡闹钟 |
| 闹钟 | `alarm:dismiss` | 取消闹钟 |
| 设置 | `settings:get` | 获取设置 |
| 设置 | `settings:update` | 更新设置 |
| 数据 | `data:export` | 导出数据 |
| 数据 | `data:import` | 导入数据 |

### 3.2 预加载模块 (Preload)

预加载脚本通过 `contextBridge` 安全地暴露 API 给渲染进程。

#### 3.2.1 API 结构

```typescript
// src/preload/index.ts
const api = {
  event: eventAPI,     // 事件 CRUD
  alarm: alarmAPI,     // 闹钟操作
  settings: settingsAPI, // 设置管理
  data: dataAPI        // 数据导入导出
}

// 安全暴露
contextBridge.exposeInMainWorld('api', api)
```

#### 3.2.2 类型安全

所有 API 接口在 `src/preload/types/index.ts` 中定义，确保编译时类型检查。

## 4. 前端模块 (Renderer)

### 4.1 状态管理 (Pinia Stores)

#### 4.1.1 日历状态 (calendar.store.ts)

**管理内容**：
- `currentDate`: 当前查看的日期
- `selectedDate`: 选中的日期
- `viewMode`: 视图模式 ('month' | 'week' | 'day')
- `selectedTimeSlot`: 选中的时间段

**核心方法**：
- `nextMonth()` / `prevMonth()`: 月份导航
- `nextWeek()` / `prevWeek()`: 周导航
- `nextDay()` / `prevDay()`: 日导航
- `goToToday()`: 返回今天
- `getMonthDays()`: 获取月视图日期数组
- `getWeekDays()`: 获取周视图日期数组

#### 4.1.2 事件状态 (event.store.ts)

**管理内容**：
- `events`: 事件列表
- `loading`: 加载状态
- `error`: 错误信息

**核心方法**：
- `fetchEvents()`: 获取事件列表
- `createEvent()`: 创建事件
- `updateEvent()`: 更新事件
- `deleteEvent()`: 删除事件
- `searchEvents()`: 搜索事件
- `getEventsByDate()`: 获取某日事件

#### 4.1.3 闹钟状态 (alarm.store.ts)

**管理内容**：
- `alarms`: 闹钟列表
- `activeAlarm`: 当前触发的闹钟
- `loading`: 加载状态

**核心方法**：
- `fetchAlarms()`: 获取闹钟列表
- `snoozeAlarm()`: 贪睡闹钟
- `dismissAlarm()`: 取消闹钟
- `setupAlarmListener()`: 设置闹钟触发监听

#### 4.1.4 设置状态 (settings.store.ts)

**管理内容**：
- `settings`: 应用设置对象
- `loading`: 加载状态

**核心方法**：
- `fetchSettings()`: 获取设置
- `updateSettings()`: 更新设置
- `applyTheme()`: 应用主题

#### 4.1.5 对话状态 (conversation.store.ts)

**管理内容**：
- `sessions`: 对话会话列表
- `currentSessionId`: 当前会话 ID
- `isListening`: 是否正在录音
- `isProcessing`: 是否正在处理
- `transcribedText`: 语音转文字结果
- `crudCountdownTimer`: CRUD 倒计时定时器

**核心方法**：
- `startListening()`: 开始语音录制
- `stopListening()`: 停止语音录制
- `processTextInput()`: 处理文本输入
- `executeCRUD()`: 执行 CRUD 操作
- `startCRUDCountdown()`: 开始自动执行倒计时
- `abandonCRUD()`: 放弃当前操作

## 6. AI 集成详解

### 6.1 语音识别流程

```
用户说话 → MediaRecorder 录制 → WebM/Opus → WAV 转换 → ASR API → 文本
```

**技术细节**：
1. 使用 `navigator.mediaDevices.getUserMedia` 获取麦克风
2. MediaRecorder 支持多种 MIME 类型（webm/ogg/mp4）
3. 音频转换为 WAV 格式（16kHz，单声道）
4. 调用智谱 ASR API 进行转写

**配置参数**：
```typescript
const audioConfig = {
  echoCancellation: true,      // 回声消除
  noiseSuppression: true,      // 降噪
  channelCount: 1,             // 单声道
  sampleRate: 16000            // 采样率
}
```

### 6.2 自然语言理解流程

```
文本输入 → LangGraph 工作流 → LLM 解析 → JSON 提取 → CRUD 操作列表
```

**工作流节点**：

1. **Preprocess（预处理）**
   - 输入校验
   - 空输入过滤

2. **LLM Parse（LLM 解析）**
   - 调用配置的 LLM API
   - 使用系统提示词引导解析
   - 返回 JSON 格式结果

3. **Assemble（组装）**
   - 解析 LLM 输出
   - 验证操作完整性
   - 兜底处理（本地正则解析）

**系统提示词关键规则**：

```markdown
## 操作类型
- 创建/添加/安排/开会/约/提醒/计划 → "create"
- 改/修改/变更/调整/推迟/提前 → "update"
- 删除/取消/去掉/移除/不要了 → "delete"

## 时间
- 相对: 明天=+1天, 后天=+2天, 下周X=下一个X, 下个月=+1月
- 上午→09:00, 下午→14:00, 晚上→19:00

## 置信度
- 信息完整→0.9-1.0, 部分缺失→0.6-0.8, 猜测→0.3-0.5
```

### 6.3 CRUD 自动执行

**流程**：
```
文本解析 → 显示操作列表 → 倒计时开始 → 自动执行 OR 用户选择 → 结果反馈
```

**配置项**：
- `crudCountdown`: 倒计时秒数（默认 10 秒）
- `crudAutoExecute`: 执行策略
  - `best`: 自动执行最高置信度方案
  - `manual`: 手动选择后执行
  - `abandon`: 超时自动放弃

## 10. 常见问题

### 10.1 语音识别不工作

**检查项**：
1. 是否配置了智谱 API Key
2. 麦克风权限是否授予
3. 网络连接是否正常

### 10.2 AI 解析不准确

**优化建议**：
1. 使用更明确的指令（如"明天下午3点"而非"明天下午"）
2. 检查 API Key 和模型配置
3. 查看日志中的 LLM 响应

### 10.3 闹钟不触发

**检查项**：
1. 应用是否在后台运行
2. 系统通知权限是否开启
3. 闹钟时间是否已过
