# 前置信息

本项目是一款**纯客户端、无后端服务**的智能语音日历闹钟系统，基于Electron框架开发，适配Windows桌面平台。，集成完整时间展示、节假日查询、日程管理、闹钟提醒、智能对话解析、语音交互能力，支持用户自主配置语音识别、大语言模型（LLM）等核心AI参数，实现高度个性化适配。

系统核心特色为**离线快速翻译、在线异步校验、自动降级**的双模式架构，无需部署后端服务，所有核心逻辑、数据存储、AI推理均在本地客户端完成。支持用户通过语音、文字对话两种方式输入日程信息，可智能识别纯文本课程表、口语化日程指令，自动完成日程批量增删改查（CRUD）操作，同时具备毫秒级闹钟提醒、时间可视化展示、农历节气查询、法定节假日识别等完整日历工具能力。

# 阶段1：基础日历

**目标**：构建一个稳定、完整的桌面日历工具，支持手动创建/编辑/删除事件、多视图浏览、节假日/农历显示、闹钟提醒、数据导入导出。此阶段无 AI 能力，但架构预留 AI 接入点。

### 2.1 事件（Event）模型

```typescript
interface CalendarEvent {
  id: string;                    // 唯一标识，UUID v4
  title: string;                  // 事件标题
  description?: string;           // 事件描述（可选）
  startDate: string;             // 开始日期，ISO 8601 格式
  endDate: string;               // 结束日期，ISO 8601 格式
  startTime?: string;            // 开始时间，HH:mm 格式（整天事件可为空）
  endTime?: string;              // 结束时间，HH:mm 格式
  isAllDay: boolean;             // 是否为全天事件
  isLunar: boolean;              // 是否使用农历日期
  repeat?: RepeatRule;          // 重复规则（可选）
  alarm?: AlarmRule;            // 闹钟规则（可选）
  category: EventCategory;      // 事件分类
  color?: string;               // 自定义颜色（可选）
  location?: string;            // 地点（可选）
  createdAt: string;             // 创建时间
  updatedAt: string;             // 更新时间
}

type EventCategory = 'work' | 'personal' | 'holiday' | 'important' | 'custom';

interface RepeatRule {
  frequency: 'daily' | 'weekly' | 'monthly' | 'yearly';
  interval: number;              // 重复间隔
  endDate?: string;             // 重复结束日期
  count?: number;               // 重复次数
  weekdays?: number[];          // 每周重复的星期（0-6）
  monthDay?: number;            // 每月重复的日期
}

interface AlarmRule {
  id: string;
  before: number;               // 提前提醒分钟数
  isEnabled: boolean;
  sound?: string;              // 提醒声音（可选）
```

### 2.4 数据存储方案

**存储位置**：`userData` 目录（由 Electron 自动管理）

**存储技术**：使用 `electron-store` 进行本地数据持久化

**存储架构**：

- **主数据文件**：`valendar-data.json` - 包含所有事件、设置和闹钟数据
- **备份目录**：`backups/` - 自动备份目录，最多保留5个备份

**数据结构**：

```json
{
  "events": [...],
  "settings": {...},
  "alarms": [...]
}
```

**数据安全机制**：

1. **自动备份**：在创建、更新、删除事件前自动创建备份
2. **数据验证**：事件创建和更新时进行输入验证
3. **备份管理**：自动清理旧备份，保留最近5个
4. **备份恢复**：支持从备份文件恢复数据

**事件输入验证规则**：

- 标题不能为空，长度不超过200字符
- 开始日期和结束日期不能为空
- 开始日期不能晚于结束日期
- 时间格式必须为 HH:mm
- 结束时间必须晚于开始时间（同一天内）
- 描述长度不超过2000字符

```typescript
    const delay = triggerTime - Date.now();

      this.alarms.set(alarm.id, timer);
    }
```

### 5.2 设计原则

1. **清晰直观**：日历视图占据主要空间，事件信息一目了然
2. **操作便捷**：主要操作（新建、编辑、删除）易于触达
3. **视觉反馈**：节假日、农历等信息用不同颜色区分
4. **响应式**：支持不同窗口大小自适应布局
5. **主题支持**：支持浅色/深色主题切换

### 5.3 颜色规范

采用**灰白高级感**设计语言，营造简洁、专业的视觉体验：

```css
:root {
  /* 主色调 - 灰白色系 */
  --color-primary: #2C3E50;          /* 深灰蓝 - 主色调 */
  --color-primary-light: #34495E;   /* 浅灰蓝 */
  --color-primary-hover: #1A252F;   /* 深灰蓝悬停态 */

  --color-weekend: #7F8C8D;        /* 周末 - 灰色 */
```

- 使用细腻的阴影效果增加层次感

## 6. 开发规范

### 6.1 代码规范

- **TypeScript**：严格模式开启，所有类型必须显式声明
- **Vue 组件**：使用组合式API（Composition API）+ `<script setup>`
- **命名规范**：
  - 组件名：PascalCase（如 `EventCard.vue`）
  - 方法名/变量名：camelCase
  - 常量：UPPER\_SNAKE\_CASE
  - CSS 类名：kebab-case

### 6.2 Git 提交规范

```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
chore: 构建/工具相关
```

## 7. 性能优化考虑

### 7.1 渲染性能

- 使用 `Virtual Scrolling` 处理大量事件
- 事件列表懒加载
- 日历视图按需渲染（只渲染可见区域）

### 7.2 数据处理

- 事件按日期索引，快速查询
- 农历计算缓存，避免重复计算
- 节假日数据本地缓存

### 7.3 内存管理

- 及时清理定时器
- 避免内存泄漏
- 监控应用内存使用
